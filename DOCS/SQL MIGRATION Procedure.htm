SQL MIGRATION Procedure

Doc ID: {89BF1C53-5EEE-472a-8BDF-DCC53A995685}

 

Before Migration FiP is launched and closed to be sure that it is properly installed.

Migration SQL db procedure:

 

1.       Database is dropped if exists

2.       Database is restored from corresponding file and all FIP users passwords are cleaned using corresponding SQL query

3.       Fieldpro is launched

4.       If corresponding DSN name exists – it is deleted

5.       New DSN name is created that connects to the database restored

6.       Corresponding DSN name is selected on login screen

7.       User name is set to Administrator

8.       Launch button is pressed

9.       On appearing migration screen “Migrate” button is pressed

10.   Waiting for migration process to be finished

11.   Entering FIP

12.   Checking system messages and if “DB Error” string can be found  - assuming error in migration

 

Path to Folder with DB can be indicated in command line

By default All DB should be located in “D:\BST_4_SQL_SERVER\MIGRATIONS\”
